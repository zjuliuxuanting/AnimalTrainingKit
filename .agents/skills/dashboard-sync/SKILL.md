# Dashboard Sync — 管理层状态同步

你是管理层（Codex Opus），项目状态发生任何变化后，必须同步以下 **5 个文件**：

---

## 同步清单

| 优先级 | 文件 | 更新内容 |
|:---:|------|------|
| **1** | `software/project-dashboard.html` | 角色状态卡片、Bug 状态、V0.5 路线图进度、活动日志卡片 |
| **2** | `项目/00_活动日志.md` | 新增事件行（日期 + 角色 + 活动） |
| **3** | `项目/04_验收与测试/Bug跟踪.md` | Bug 状态更新、新 Bug 登记 |
| **4** | `~/.Codex/projects/*/memory/project_progress.md` | 版本门禁、阻塞项、当前代码状态 |
| **5** | `角色/AI/{PM,软件工程师,测试}/memory.md` | 仅当该角色的"当前状态"或"下一动作"改变时更新 |

---

## 触发条件

收到以下任何消息时，自动执行同步：
- 软工交付（deliver 清单 + 测试结果）
- 测试完成（测试报告）
- 创始人做决策（Bug 纳入/排除、方案选择）
- 门禁状态变更（开门/关门）
- 新 Bug 发现

---

## 执行规则

1. **完成眼前任务后再同步** — 不要打断当前对话
2. **并行更新前 3 个文件** — 仪表盘 + 活动日志 + Bug 跟踪可同时写
3. **第 4 个文件（memory）如有明显状态变化才更新** — 小改动不配更新 memory
4. **第 5 个文件（角色 memory）仅在角色状态有实质性改变时才更新** — 比如从"待命"变"完成"
5. **同步完要告知用户** — 一句话列出改了哪些文件

---

## 同步前检查

执行同步前，先用以下命令确认当前状态：
```bash
# 检查哪些文件最近变了
ls -la software/project-dashboard.html 项目/00_活动日志.md 项目/04_验收与测试/Bug跟踪.md
```

---

## 失败自检

如果用户说"面板更新了么"或"XX 文档没更新"，说明同步流程漏了。立即补更，并在活动日志里加一条：`管理层 | 🔧 补更：仪表盘/活动日志同步遗漏`。
