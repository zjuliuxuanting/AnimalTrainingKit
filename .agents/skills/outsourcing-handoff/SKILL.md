# Outsourcing Handoff — 外包对接管理

## 一、对外包方的定位

外包方是**端到端交付方**，负责从方案设计到实物交付的整个过程。

甲方的角色是**需求提出方和验收方**，不是技术协作者。

## 三、对接会议标准议程

### 第一阶段：需求对齐（约 40 分钟）

1. 甲方讲实验场景和痛点（10min）
2. 甲方带读 PRD（15min）
3. 外包方提问，澄清需求（15min）

### 第二阶段：技术方案讨论（约 30 分钟）

1. 外包方初步判断可行性
2. 外包方提出技术方案方向（不要甲方替他想）
3. 确定：哪些技术细节外包方自定，哪些甲方有硬约束

### 第三阶段：合作框架（约 20 分钟）

1. 过项目任务书的里程碑和交付物
2. 外包方给出初步排期和报价
3. 约定下一步：方案文档提交时间

## 四、常见 Q&A（外包方可能问的问题）

| 问题 | 建议回答 |
|------|---------|
| "主控用什么芯片？" | "你根据功能需求自己定，能满足我的要求就行" |
| "通信用 Wi-Fi 还是蓝牙？" | "你看哪个更合适，我不限定" |
| "连接器用什么型号？" | "你来选，保证稳定可靠就行" |
| "软件用什么语言写？" | "你自己熟悉的技术栈，能交付 Windows 可运行的程序就行" |
| "协议怎么定义？" | "你来设计，保证跟我软件通信稳定就行" |
| "预算多少？" | 根据实际情况回答 |
