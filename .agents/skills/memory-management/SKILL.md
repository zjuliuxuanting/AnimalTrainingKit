# Memory Management — AI 记忆管理

## 三层记忆架构

本项目的 AI 记忆分为三层，每层的持久化程度和范围不同：

### 第一层：长期记忆（项目规则）

| 属性 | 说明 |
|------|------|
| **存储位置** | `AGENTS.md`（项目根目录） |
| **持久性** | 永久，跨所有会话 |
| **内容** | 项目基本信息、角色设定、核心原则、沟通风格 |
| **更新频率** | 低（只有项目重大变化时才改） |

### 第二层：中期记忆（角色工作记忆）

| 属性 | 说明 |
|------|------|
| **存储位置** | `角色/AI/<角色名>/memory.md` |
| **持久性** | 跨会话，手动更新 |
| **内容** | 当前阶段状态、关键决策、阻塞项、经验教训、常用路径 |
| **更新频率** | 中（每周更新，或阶段变更时更新） |

已有文件：
- [角色/AI/PM/memory.md](../../角色/AI/PM/memory.md) — PM 工作记忆
- [角色/AI/软件工程师/memory.md](../../角色/AI/软件工程师/memory.md) — 软件工程师工作记忆
- [角色/AI/测试/memory.md](../../角色/AI/测试/memory.md) — 测试工程师工作记忆

### 第三层：短期记忆（会话上下文）

| 属性 | 说明 |
|------|------|
| **存储位置** | 当前对话上下文（不持久化到文件） |
| **持久性** | 仅当前会话 |
| **内容** | 本轮对话的细节、讨论中的决定、临时计算 |
| **更新频率** | 高（每次对话） |

### 第四层：持久化记忆（Codex 内置）

Codex 在 `~/.Codex/projects/<project-path>/memory/` 提供额外的持久化记忆系统，用于存储用户偏好、反馈、项目参考等信息。

## 记忆更新规则

### 什么时候该更新

| 触发条件 | 更新哪层 | 更新什么 |
|---------|---------|---------|
| 项目阶段变更 | 长期 + 中期 | 阶段状态、里程碑 |
| 关键决策拍板 | 中期 | 决策记录、阻塞项状态 |
| 风险状态变化 | 中期 | 风险登记册 |
| 用户偏好变化 | 长期 | 沟通风格、规则 |
| 对话结束时的关键信息 | 中期 | 重要结论、待办 |

### 更新原则

1. **先查重**：更新前检查是否有相关内容已存在，避免重复
2. **追加 vs 覆盖**：经验教训类追加，状态信息类覆盖
3. **保持精简**：每条记忆不超过 200 字，关键信息优先
4. **文件头标注日期**：每次更新修改"最后更新"时间戳
