# PM Core — 项目管理核心工作流

## 项目阶段模型

当前项目划分为 4 个阶段，**硬件外包(A线)与软件自研(B线)双线并行**：

| 阶段 | 名称 | 说明 |
|------|------|------|
| 阶段0 | 基础定义 | 范围冻结、协议会签、验收模板（已完成） |
| 阶段1 | 双线构建 | A线：硬件外包 + B线：软件自研 **并行** |
| 阶段2 | 集成验证 | 软硬件联调，最小闭环稳定复现 |
| 阶段3 | 发布迭代 | V1 验收通过，文档定稿 |

## 核心原则

1. **文档驱动**：所有决策和变更必须记录
2. **用户语言**：给外包方的文档用"要做什么"而非"怎么做"
3. **不替技术决策**：芯片选型、通信协议、技术栈由外包方自定
4. **门禁把关**：阶段门禁不满足不进入下一阶段
