# Context Sync — 跨会话上下文同步

## 会话启动流程

每次新会话开始时，按以下顺序加载上下文：

```
Step 1: 加载长期记忆
  读取 AGENTS.md（项目根目录）
  获取：角色设定、核心原则、沟通风格

Step 2: 加载当前角色中期记忆
  读取 角色/AI/<当前角色>/memory.md
  获取：当前阶段状态、关键决策、阻塞项、经验

Step 3: 加载项目索引
  读取 项目/00_索引与总览.md
  获取：文件索引、系统架构速查、核心协议速查

Step 4: 加载当前阶段看板
  读取 项目/01_阶段规划/推进看板.md
  获取：当前任务、阻塞项、下一步动作
```

## 会话结束（或关键节点）同步

当以下情况发生时，考虑更新记忆：

### 需要更新的场景

| 场景 | 更新内容 | 更新位置 |
|------|---------|---------|
| 用户做了关键决策 | 决策内容 + 日期 | 决策记录.md + memory.md |
| 阶段状态变更 | 新阶段 + 新目标 | memory.md + 推进看板 |
| 风险状态变化 | 风险状态 + 缓解措施 | 风险登记册.md |
| 文档发生重大修改 | 文件状态变更 | 00_索引与总览.md |
| 用户反馈了偏好 | 偏好内容 | AGENTS.md |
| 发现了重要经验 | 经验内容 | memory.md |

### 更新检查清单

- [ ] 本次对话是否有新决策？→ 写入决策记录
- [ ] 是否有风险状态变化？→ 更新风险登记册
- [ ] 是否有文档变更？→ 更新索引状态
- [ ] 是否有经验教训？→ 追加到 memory.md
- [ ] 是否有待办事项？→ 更新推进看板

## 冲突处理

如果新旧信息矛盾：

1. **以用户最新指示为准**：用户明确说了"改"，就按新的来
2. **记录变更理由**：在决策记录中注明"X年X月X日，用户指示从A改为B，原因：..."
3. **同步更新所有关联文件**：确保索引、看板、memory 之间不矛盾
